# LLM Teaching Notebooks

This repository is a teaching package for explaining how large language models are built, trained, and applied.

It is organized as three Jupyter notebooks that move from first principles to modern Hugging Face workflows:

1. `notebooks/01_mini_transformer_from_scratch.ipynb`
2. `notebooks/02_attention_encoder_decoder_seq2seq.ipynb`
3. `notebooks/03_finetuning_and_applications_with_hf.ipynb`

The material is designed for upper-level college students who are already comfortable with PyTorch and want an intuitive, visualization-heavy walkthrough of transformer-based NLP.

## What This Repo Covers

- conceptual overview of transformers and LLMs
- tokenization, embeddings, and positional information
- attention, softmax, and causal masking
- decoder-only transformers from scratch
- encoder-decoder transformers and sequence-to-sequence learning
- LLM memory visualization
- fine-tuning pretrained transformers such as BERT, GPT-style models, and T5
- applications including sentiment analysis, summarization, and chatbot-style generation

## Setup

These setup instructions are for Windows PowerShell.

The recommended setup path is the repo bootstrap script:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\setup_windows.ps1
```

That script will:

- create `llm-create-demo` if it does not already exist
- detect whether `nvidia-smi` is available
- install a CUDA-enabled PyTorch build on NVIDIA Windows machines
- fall back to a CPU build if no NVIDIA GPU is detected
- install the rest of the Python dependencies
- register the Jupyter kernel

If you prefer to run the steps manually:

```powershell
python -m venv llm-create-demo
llm-create-demo\Scripts\Activate.ps1
python -m pip install --upgrade pip
python .\scripts\bootstrap_env.py
```

## Register the Jupyter Kernel

The setup script already registers the kernel.
If you need to run that step manually after the virtual environment is active:

```powershell
python -m ipykernel install --user --name llm-create-demo --display-name "Python (llm-create-demo)"
```

You can verify that Jupyter sees it with:

```powershell
jupyter kernelspec list
```

If you ever need to remove and recreate the kernel:

```powershell
jupyter kernelspec uninstall llm-create-demo
```

## Launch Jupyter

Start Jupyter after the environment is active:

```powershell
jupyter lab
```

or

```powershell
jupyter notebook
```

Inside Jupyter, select the kernel named `Python (llm-create-demo)`.

Notes:

- `requirements.txt` no longer installs PyTorch directly. PyTorch is installed by `scripts/bootstrap_env.py` so the repo can choose a CUDA-enabled build when an NVIDIA GPU is present.
- Each notebook also contains an install cell so the environment setup is visible to students during class.

## Recommended Teaching Order

### 1. Mini Transformer From Scratch

Use this notebook to show the internal pipeline of a decoder-only LLM:

- raw text loading
- character tokenization
- embeddings
- scaled dot-product attention
- causal masking
- transformer blocks
- cross-entropy training
- text generation and context-window memory visualization

### 2. Encoder-Decoder Attention and Seq2Seq

Use this notebook to explain why attention matters in sequence-to-sequence models:

- bottlenecks in old seq2seq designs
- encoder self-attention
- decoder self-attention
- cross-attention
- summarization with T5 on SAMSum

### 3. Fine-Tuning and Applications With Hugging Face

Use this notebook to connect the theory to practical workflows:

- BERT-style text classification on IMDb
- GPT-style dialogue modeling on DailyDialog
- chatbot-style generation
- summarization applications

## Checkpoints and Class-Time Control

The notebooks are designed to support both:

- live training during class
- checkpoint loading when time is tight

Saved models are stored under `artifacts/checkpoints/`.

Suggested workflow before class:

1. Run the notebooks once on the instructor machine.
2. Let the training sections save local checkpoints.
3. In class, keep checkpoint loading enabled so timing stays predictable.

## Regenerating the Notebooks

The notebooks in `notebooks/` are generated by:

```powershell
python scripts/generate_teaching_notebooks.py
```

If you want to revise markdown, code cells, or notebook structure in one place, edit the generator script and regenerate the notebooks.

## Supporting Files

- `INSTRUCTOR_GUIDE.md` contains pacing suggestions, dataset rationale, and checkpoint strategy.
- `requirements.txt` contains the non-PyTorch Python dependencies for the teaching environment.
- `scripts/bootstrap_env.py` chooses the correct PyTorch build for the current Windows machine.
- `scripts/setup_windows.ps1` creates the virtual environment, installs dependencies, and registers the Jupyter kernel.

## Datasets Used

- `karpathy/tiny_shakespeare`
- `knkarthick/samsum`
- `stanfordnlp/imdb`
- `li2017dailydialog/daily_dialog`

These were chosen because they are teaching-friendly, readable in class, and small enough to support local-GPU demos.
